第一部分: 扩展语法

Markdown 的基础语法无法满足所有需求，扩展语法应运而生。扩展语法通过添加表格、代码块、脚注等元素来丰富 Markdown 的功能。

（一）表格

使用|分隔列，使用 ---- 定义表头和内容的分隔线

使用 : 来设置内容对齐方式，如: :---- 表示左对齐， ----: 表示右对齐， :----:表示居中对齐

⚠️ 提示：

  1. 表格中可以设置文本格式；如：加粗，斜体，行内代码

  2. 表格中可以使用HTML标签

  • 内容对齐示例:

| 左对齐表头 | 居中对齐表头 | 右对齐表头 | |:------ |:------:|-------:| | 1 | 2 | 3 | | 4 | 5 | 6 |

（二）代码块

  • 行内代码 ：使用反引号 ` 包裹代码。

  • 围栏代码块

使用三个反引号 ``` 创建代码块， 在反引号后指定代码语言，可实现语法高亮。

``` json { "name": "数说编程", "age": 22, "gender": "男" } ```

（三） 删除线

请在单词前后使用两个波浪号 ~~。

渲染效果如下:

会议时间： 下午3点 → 上午10点

（四）任务列表

任务列表语法：使用 [ ] 表示未完成， [x] 表示已完成。

- [ ] 未完成的任务 - [x] 已完成的任务（打勾）

渲染效果如下:

  • [ ] 未完成的任务

  • [x] 已完成的任务（打勾）

（五）脚注

脚注是文档中用于提供补充信息或引用来源的一种标记方式。在 Markdown 中，脚注允许你为文档中的特定内容添加详细说明或参考资料，而不会打断正文的阅读流程。

⚠️ 提示： 不同平台对脚注的支持程度不同，使用前请确认目标平台是否支持

脚注语法：

[文字](脚注解释 "脚注名字")

示例：

[Markdown](https://daringfireball.net/projects/markdown/ "什么是Markdown")在业务开发流程中起到了至关重要的作用。 [Markdown是什么](是一种轻量级的标记语言。 "什么是Markdown")在业务开发流程中起到了至关重要的作用。

渲染效果如下:： 请拉到最下面观看

Markdown在业务开发流程中起到了至关重要的作用。

Markdown是什么在业务开发流程中起到了至关重要的作用。

（七）TOC（Table of Contents）

TOC 是自动生成目录的功能，会根据文档标题层级创建导航目录。

支持平台：微信公众号、知乎。
注意：微信平台仅支持二级和三级标题显示

TOC 语法：

[TOC]

渲染效果如下:

[TOC]

（八）注音符号

支持平台：微信公众号。

支持注音符号，用法如下：

欢迎关注我的公众号，{数说编程|shù shuō biān chéng}！

渲染效果如下:

欢迎关注我的公众号，{数说编程|shù shuō biān chéng}！

第二部分: 变通语法

我们除了能使用扩展语法，还可以使用 HTML 变通方法实现特殊效果。

⚠️ 注意：HTML标签兼容性因平台而异

（一） 下划线

一些文字 <ins>将被加下划线</ins>。

渲染效果如下:

一些文字 。

（二） 缩进（Tab）

&nbsp;&nbsp;&nbsp;&nbsp;这是一个缩进的段落。

渲染效果如下:

这是一个缩进的段落。

（三） 文字居中

<p style="text-align:center">这段文字居中显示。</p>

渲染效果如下:

这段文字居中显示。

（四） 文字颜色

<font color="red">这段文字是红色的！</font> <p style="color:blue">这段文字是蓝色的。</p>

渲染效果如下:

这段文字是红色的！

这段文字是蓝色的。

（五） 隐藏注释

这是可见的段落。 [这是一个隐藏的注释]: # 这是另一个可见的段落。

渲染效果如下:

这是可见的段落。

这是另一个可见的段落。
